2. 3 reasons why D3 is helpful: (1) It allows the users to interact with the graph to see what happens with different data sets. Users can change data set by clicking on buttons or selecting options which and the same graph wil modify accordingly to the new data, which gives the user a direct comparison. (2) D3 enables many types of graphs. Not only the basic bar/pie chart, but also violin, density, correlation, spider/radar and even geographic map. All these helps to analyze one dataset from more perspectives and dimentions, so the user can make the most out of the informaiton from the data. (3) D3 also allows us to perform statistical tests. It can be helpful when combined with statistics, such as t test and regressions with scatter plot. When calculating the regression, for example the best fit-line and the data can be compared side by side.

3. 3 reasons why D3 may not be the best visualization tool: (1) D3 requires the users to know what to do with the data. As opposed to excel or STATA, which does the chart and regression automatically for the users when the data is inputed, D3 requires the user to hand draw the rectangles for the bar chart and the circle for pie chart. So it can be confusing when the user don't know what to do with the data. (2) There aren't many built-in stats tests portion for D3. So it is not easy to calculate the statistical values such as "R^2" or perform a t-test/f-test. (3) Because D3 works more with the DOM and on decoration of the graphs, less with the data itself, users may not get a deep understanding of the data and how it is visualized.
